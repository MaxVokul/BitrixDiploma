Understanding ($counter - 1) % 3 != 2

What does this expression mean?

This expression is used to check if we need to close a row div in a grid layout after processing items in a loop.

($counter - 1) % 3 != 2

Let's break it down:

  • $counter - A variable that tracks the current position in the loop (starting from 0)
  • % - The modulo operator, which returns the remainder after division
  • 3 - We're grouping items in sets of 3
  • != 2 - Checking if the result is NOT equal to 2

In simpler terms: This condition checks if the previous item (counter-1) was NOT the last item in a group of three.

If the condition is true, it means we need to close the row because we've either completed a full row or reached the end of the items.

// Practical usage in a loop:
$counter = 0;
foreach ($items as $item) {
  // Process item...
  
  // Check if we need to close the row
  if (($counter - 1) % 3 != 2) {
    echo '</div>'; // Close the row
  }
  
  $counter++;
}

Visualization with Examples

Let's see how this expression evaluates for different counter values:

Counter Counter - 1 (Counter - 1) % 3 Result != 2 Meaning
0 -1 2* False Don't close row
1 0 0 True Close row
2 1 1 True Close row
3 2 2 False Don't close row
4 3 0 True Close row
5 4 1 True Close row
6 5 2 False Don't close row

* Note: In PHP, negative modulo operations work differently than in some other languages. -1 % 3 equals 2 in PHP.

When is this condition True?

The condition is TRUE when we need to close a row, which happens after processing:

  • The 1st item (if we only have 1 item)
  • The 2nd item (if we only have 2 items)
  • The 4th item (if we only have 4 items)
  • The 5th item (if we only have 5 items)
  • And so on...

The condition is FALSE when we've just finished a complete row of 3 items and don't need to close a row prematurely.
